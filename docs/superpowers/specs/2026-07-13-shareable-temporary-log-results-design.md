# 可分享临时日志结果设计

## 目标

用户可以针对当前打开的数据集输入包含 `AND`、`OR`、`NOT`、括号和引号短语的表达式，预览匹配结果，并将完整原始日志行重定向为可分享、可下载、可继续搜索的临时日志文件。

## 数据范围

- 普通文件视图：数据集是当前文件的全部内容。
- Issue 内容搜索视图：数据集是当前搜索命中的文件集合，并保留原主搜索作为基础约束。
- 临时结果视图：数据集是该临时文件，可以继续过滤并生成新的临时结果。

首版在 UI 中从已打开文件创建临时结果；接口的数据模型保留多个源文件的来源字段，为跨文件结果重定向提供相同输出格式。

## 表达式

- 操作符大小写不敏感：`AND`、`OR`、`NOT`。
- 优先级：`NOT` 高于 `AND`，`AND` 高于 `OR`。
- 支持括号和双引号短语。
- 普通词与短语均进行大小写不敏感的 substring 匹配。
- 相邻条件没有操作符时返回带位置的语法错误，不隐式猜测。

## 临时结果

- 后端流式读取源文件并写入匹配的完整原始行，避免大文件常驻内存。
- 多文件输出在每个来源前写入 `# source: <path>`；单文件也保留来源标记，确保分享后可追溯。
- 输出按来源顺序和原始行号排序，并去重。
- 元数据存入 SQLite，文件写入数据目录下的 `temp-results/`。
- UUID 作为不可猜测的分享 ID。
- 默认 7 天过期；每次读取、分页、搜索或下载后续期 7 天。
- 创建者可手动删除；过期记录在访问时惰性清理。
- 服务重启后仍有效，不进入 Issue 文件树和正式索引流程。

## API 与 UI

- `POST /api/temp-results`：从当前文件和表达式生成临时结果。
- `GET /api/temp-results/{id}`：读取元数据。
- `GET /api/temp-results/{id}/lines`：分页查看。
- `GET /api/temp-results/{id}/download`：下载。
- `DELETE /api/temp-results/{id}`：删除。
- 文件搜索框支持表达式，并增加轻量“生成临时文件”按钮。
- 生成后进入独立临时结果视图，显示来源、表达式、到期时间、复制分享链接、下载和删除。

## 验证

- 单元测试覆盖解析优先级、括号、短语、NOT 和错误位置。
- 集成测试覆盖生成、完整行、分享读取、分页、下载、删除和文件隔离。
- 前端执行 TypeScript/Vite 构建；后端执行完整 Rust 测试。
