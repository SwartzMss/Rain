# 搜索结果日志文件式显示设计

## 目标

详细日志搜索和文件内搜索的结果重新以普通日志文件的形式连续显示。用户浏览或选择日志文本时不会触发跳转；只有在需要追溯来源时，才通过日志行的右键菜单打开原文件并定位到对应行。

## 交互

- 搜索结果采用与文件查看器一致的“行号 + 日志内容”连续布局，不再把每条命中显示成独立卡片。
- 左键不执行来源跳转，浏览器原生的文本选择行为保持可用。
- 在具备来源元数据的日志行上右键时，显示 Rain 自定义菜单。
- 右键菜单只包含“在原文件中打开”。不再提供“复制文件路径”。
- 执行菜单项后保留搜索结果页签，打开或激活对应原文件页签，并滚动、高亮目标行。
- 点击菜单外、按 Escape、滚动结果区或执行菜单项后关闭菜单。

## 数据与组件

搜索响应和查看器页签继续保留 `bundle_hash`、`file_id`、`path` 与 `line_number`，这些字段只用于右键来源定位和必要的辅助信息，不改变后端接口。

`SearchResultViewer` 使用连续的双列日志布局渲染当前页结果。每个日志内容行承载右键事件，但不使用按钮语义或左键处理器。`SearchHitContextMenu` 删除复制路径相关属性和菜单项，只负责触发来源打开。

`BundleView` 继续复用现有的来源解析、文件页签和目标行定位流程；删除不再使用的复制路径处理逻辑。分页、结果内继续搜索和命中关键词高亮保持不变。

## 异常与边界

- 缺少来源文件元数据的行仍可正常阅读和选择文本；右键不显示无效的定位菜单。
- 缺少原始行号但文件来源有效时，打开文件第一页，并沿用现有提示。
- 原文件已删除或加载失败时，保留当前搜索结果并显示现有错误提示。
- 行号栏展示原始文件的一基行号；来源行号不可用时展示当前结果页内的一基顺序号，且该回退编号不用于原文件定位。
- 搜索结果分页不会改变原始来源元数据。

## 验证

- 测试搜索结果采用连续日志布局，不再渲染可左键打开的按钮或卡片。
- 测试左键不触发来源打开，日志文本可选择。
- 测试有来源的行右键出现仅含“在原文件中打开”的菜单，并传递正确命中信息。
- 测试缺少来源的行不提供定位菜单。
- 测试菜单的 Escape、外部点击和滚动关闭行为继续有效。
- 运行前端测试、TypeScript 检查和 Vite 构建。
