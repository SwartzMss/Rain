# 搜索命中反向定位原文件设计

## 目标

用户在详细日志搜索或临时结果中单击一条命中后，Rain 保留当前搜索结果页签，并打开或激活对应的原文件页签，将目标行滚动到查看区中部并整行高亮。搜索命中行同时提供精简右键菜单，支持打开原文件和复制文件路径。

## 范围

- 覆盖带有 `bundle_hash`、`file_id`、`path` 和 `line_number` 来源元数据的搜索结果。
- 复用现有文件树加载、查看器页签、分段行读取和目标行定位能力。
- 不增加后端解析接口，不改变搜索响应格式。
- 不在本次加入 URL 深链、分屏查看、新浏览器窗口或跨 Issue 跳转。

## 交互

搜索结果中的每一行是可交互目标。悬停时显示可点击状态和“打开原文件”的提示。

单击命中行时：

1. 校验来源元数据。
2. 按 `bundle_hash:file_id` 加载或找到文件节点。
3. 保留搜索结果页签。
4. 新建对应文件页签；如果该文件已经打开，则激活已有页签而不重复创建。
5. 读取包含 `line_number` 的文件分页。
6. 将目标行滚动到查看区中部并对整行高亮。

行号沿用后端的零基语义；界面展示时仍使用一基行号。目标行高亮保持到用户定位到其他命中或主动滚动/切换查看内容，不使用容易错过的短时闪烁动画。

在搜索命中行上右键时，仅该区域阻止浏览器默认菜单，并显示 Rain 自定义菜单：

- **在原文件中打开**：执行与单击相同的定位行为。
- **复制文件路径**：将命中的展示路径写入剪贴板，并给出轻量成功或失败提示。

点击菜单外、按 Escape、滚动结果区或执行菜单项后关闭菜单。菜单靠近指针显示，并限制在当前视口内。

## 前端结构与数据流

`SearchResultViewer` 负责渲染命中交互，但不自行加载文件。它向 `BundleView` 回调完整命中信息；`BundleView` 负责以下协调工作：

- 根据命中的 bundle 与文件 ID 解析复合节点 ID。
- 必要时通过现有文件节点接口加载节点。
- 调用现有 `handleNodeClick` 和查看器页签逻辑。
- 将命中的原始行号设置为文件页签的 `targetLine`。

右键菜单作为独立的小型组件或明确隔离的视图状态存在，接收屏幕坐标、路径以及打开回调。剪贴板写入封装成可测试函数；优先使用 `navigator.clipboard.writeText`，不可用或被拒绝时显示错误提示，不静默失败。

现有 `CodeLinesPane` 的目标行样式扩展为明显但不遮挡文本的整行背景色。现有目标行滚动逻辑确保定位后行位于可视区中部；若当前实现只保证可见，则调整为 `block: "center"`。

## 异常与边界

- 缺少 `bundle_hash` 或 `file_id`：该命中保持可阅读，但禁用原文件打开，并提示“来源文件信息不可用”。复制路径仍可用。
- 缺少行号：打开文件第一页，不做行定位，并提示“已打开文件，原始行号不可用”。
- 文件已被删除或 bundle 不再存在：保留搜索结果页签，显示加载错误，不创建失效文件页签。
- 文件为二进制或不可预览：打开现有二进制信息页签，不尝试定位行。
- 同一文件连续定位：复用页签并更新 `targetLine`，确保第二次点击同样能触发滚动。
- 页签中的搜索结果翻页后，交互使用当前页每条命中携带的来源元数据，不使用结果页中的顺序行号代替原始行号。

## 可访问性

- 命中行使用按钮语义，可通过 Tab 聚焦并以 Enter/Space 打开。
- 右键菜单使用菜单语义，首次打开时聚焦第一项，支持上下方向键、Enter 和 Escape。
- 聚焦态与悬停态均清晰可见；提示信息通过现有可感知的状态区域展示。

## 验证

- 单元测试验证命中是否具备可打开来源、复合节点 ID 生成以及剪贴板成功/失败路径。
- 组件测试验证单击传递正确的 bundle、文件 ID 和原始行号。
- 组件测试验证右键菜单出现、视口边界限制、菜单项行为及 Escape/外部点击关闭。
- 查看器页签测试验证首次打开新建页签，再次打开同一文件只激活并更新目标行。
- 文件查看测试验证目标分页包含命中行，目标行位于可视区中部并应用高亮样式。
- 回归验证缺少来源、缺少行号、文件删除及二进制文件的行为。
- 最终运行前端测试、TypeScript 检查和 Vite 构建；后端接口不变，无需新增后端实现测试。
