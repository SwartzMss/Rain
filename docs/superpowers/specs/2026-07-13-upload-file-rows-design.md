# 上传文件行与日志页脚精简设计

## 目标

上传期间直接展示用户实际选择的文件，避免“首文件名 等 N 个文件”的汇总中间态；同时移除日志查看区重复的行范围文字，为日志内容留出更清晰的界面。

## 上传文件行

- 前端保存本次选择的每个文件名和大小，不再保存拼接后的汇总名称。
- 单文件上传立即显示一条对应文件行。
- 多文件上传立即逐条显示对应文件行，各行共享本次上传进度和处理状态。
- 后端任务出现后，乐观文件行保持稳定，直到后端返回 READY 文件列表后再替换为实际文件行。
- 压缩包上传期间显示压缩包本身；解析完成后显示解压得到的实际文件。
- 不改变上传 API、后端任务模型或失败状态展示。

## 日志页脚

- 移除“行 X - Y / Z”文字。
- 保留“下载原文件”和右下角分页控件。
- 不改变分页读取、每页条数、源文件行号、搜索结果行号或搜索跳转。

## 右侧多标签日志查看器

- 右侧内容区使用类似 Notepad++ 的标签栏，同时保留多个普通文件或搜索结果。
- 单击不同文件或执行新搜索时追加标签；再次打开同一个文件只激活已有标签。
- 标签支持固定、取消固定和关闭，关闭最后一个标签后不会被自动重新创建。
- 普通文件标签分别保存分页位置、每页条数和滚动位置。
- 日志内容搜索结果作为虚拟文件展示，只连续显示命中正文，不在每条正文前显示源文件名和源行号。
- 虚拟搜索文件按 1000/3000 行分页读取，可浏览完整匹配结果而不一次载入全部内容。
- 虚拟结果内部继续保留文件 ID、bundle hash 和真实行号，保证后续定位能力。
- 新搜索创建独立标签，便于在多个结果或文件之间直接切换。
- 生成后端临时文件后直接创建并固定一个右侧标签；该标签具备独立分享链接和下载能力。

## 验证

- 单文件和多文件的上传占位数据转换应有自动化测试。
- 前端 TypeScript 构建通过。
- 后端既有测试继续通过。
