# 精简文件树节点设计

## 目标

文件树只展示用户定位文件所需的信息，移除子节点数量、文件类型和大小等冗余元数据，同时让被截断的长文件名可以通过悬停查看完整内容。

## 节点显示

每个普通文件、目录和压缩包节点只保留：

- 展开或折叠箭头；仅可展开节点显示。
- 现有类型图标，例如目录、压缩包、二进制文件和文本文件图标。
- 单行文件名或目录名。

删除以下可见信息：

- “X 子节点”。
- MIME 类型及其他文件子类型。
- 文件大小。
- “展开”和“收起”文字。
- 空目录或空压缩包展开后的“暂无子节点”占位文字。

实际文件树结构、子文件加载和展开行为保持不变。此次改动只精简显示，不过滤或删除任何节点数据。

## 长名称

文件名继续使用单行截断，避免宽名称撑开侧栏。名称元素设置浏览器原生 `title`，鼠标悬停时显示未经截断的完整名称。节点按钮使用完整名称作为可访问名称，使键盘和辅助技术用户也能获得同样信息。

## 组件边界

改动集中在 `FileTreeNode`：移除不再使用的大小格式化和元数据计算，保留递归渲染、选中状态、缩进、图标和展开逻辑。后端接口、`TreeNode` 数据模型及文件加载流程不变。

## 验证

- 测试目录和压缩包节点不再渲染子节点数量及展开状态文字。
- 测试文件节点不再渲染 MIME 类型、子类型和文件大小。
- 测试节点仍显示名称、类型图标和展开箭头。
- 测试长名称元素带有完整名称的 `title` 和可访问标签。
- 测试展开节点仍递归渲染实际子文件，空节点不显示冗余占位信息。
- 运行完整前端测试、TypeScript 检查和生产构建。
